# 智能个人知识管理系统 - 需求分析

## 1. 引言

### 1.1 项目背景
在信息爆炸的时代，学生和知识工作者面临着信息过载的挑战。传统的笔记和文件管理方式往往导致知识孤岛，难以建立知识点之间的关联，也无法根据个人学习习惯提供智能化的学习建议。因此，开发一个智能个人知识管理系统具有重要的现实意义。

### 1.2 项目目标
开发一个基于AI技术的个人知识管理工具，帮助用户：
1. 高效管理学习资料和文档
2. 自动构建知识关联网络
3. 基于用户行为提供个性化的学习路径推荐
4. 提供可视化的知识图谱展示

### 1.3 项目范围
本项目主要面向学生和知识工作者，提供以下核心功能：
- 多格式文档管理
- 知识图谱构建与可视化
- AI智能推荐
- 学习分析与统计

## 2. 用户需求分析

### 2.1 用户角色定义

#### 2.1.1 学生用户
- **特征**: 需要管理大量学习资料，包括课件、笔记、习题等
- **需求**: 
  - 快速查找和整理学习资料
  - 建立知识点之间的联系
  - 获得个性化的学习建议
  - 跟踪学习进度

#### 2.1.2 知识工作者
- **特征**: 需要管理专业领域的知识文档和资料
- **需求**:
  - 高效的文档管理
  - 知识体系的构建和维护
  - 跨文档知识关联发现
  - 专业内容的智能处理

#### 2.1.3 研究人员
- **特征**: 需要管理研究文献和实验数据
- **需求**:
  - 文献资料的组织和检索
  - 研究思路的可视化表达
  - 研究进展的跟踪和分析
  - 协作知识的管理

### 2.2 用户场景分析

#### 2.2.1 场景一：学习资料整理
- **用户**: 大学生小李
- **情境**: 期末考试前需要复习多门课程的资料
- **需求**: 
  - 将分散在不同位置的课件、笔记、习题等资料集中管理
  - 快速找到相关知识点的资料
  - 建立课程知识点之间的联系

#### 2.2.2 场景二：知识体系构建
- **用户**: 程序员小王
- **情境**: 学习新技术框架，需要构建完整的知识体系
- **需求**:
  - 管理技术文档和教程
  - 建立技术概念之间的关系
  - 跟踪学习进度
  - 获得相关技术的推荐

#### 2.2.3 场景三：研究资料管理
- **用户**: 研究生小张
- **情境**: 进行毕业论文研究，需要管理大量文献资料
- **需求**:
  - 整理和分类研究文献
  - 提取文献核心观点
  - 建立研究思路的知识图谱
  - 跟踪研究进展

## 3. 功能需求

### 3.1 核心功能需求

#### 3.1.1 智能文档管理
- **FR-1**: 支持多种文档格式（TXT、MD、PDF、DOC、DOCX等）
- **FR-2**: 文档自动分类和标签生成
- **FR-3**: 全文检索和语义搜索
- **FR-4**: 文档版本管理
- **FR-5**: 文档预览和在线编辑

#### 3.1.2 知识图谱构建
- **FR-6**: 自动提取文档关键词和概念
- **FR-7**: 可视化知识关联网络
- **FR-8**: 支持手动创建知识链接
- **FR-9**: 知识点重要性评估
- **FR-10**: 知识图谱导出和分享

#### 3.1.3 AI智能助手
- **FR-11**: 基于用户学习习惯的内容推荐
- **FR-12**: 智能学习路径规划
- **FR-13**: 知识点掌握度评估
- **FR-14**: 个性化复习提醒
- **FR-15**: 智能问答功能

#### 3.1.4 学习分析面板
- **FR-16**: 学习时长统计
- **FR-17**: 知识点掌握度可视化
- **FR-18**: 学习进度跟踪
- **FR-19**: 知识网络成长轨迹
- **FR-20**: 学习报告生成

### 3.2 非功能需求

#### 3.2.1 性能需求
- **NFR-1**: 系统响应时间不超过3秒
- **NFR-2**: 支持同时处理10个以上并发用户
- **NFR-3**: 支持管理10万+文档
- **NFR-4**: 知识图谱渲染时间不超过5秒

#### 3.2.2 可用性需求
- **NFR-5**: 界面简洁直观，新用户5分钟内可上手
- **NFR-6**: 提供详细的操作帮助和引导
- **NFR-7**: 支持键盘快捷操作
- **NFR-8**: 提供多语言支持（中文、英文）

#### 3.2.3 可靠性需求
- **NFR-9**: 系统可用性达到99.5%
- **NFR-10**: 数据自动备份，防止丢失
- **NFR-11**: 异常情况有明确的错误提示
- **NFR-12**: 支持断点续传大文件

#### 3.2.4 安全性需求
- **NFR-13**: 用户数据本地存储，保护隐私
- **NFR-14**: 敏感数据加密存储
- **NFR-15**: 文件访问权限控制
- **NFR-16**: 操作日志记录

#### 3.2.5 兼容性需求
- **NFR-17**: 支持Windows、macOS、Linux操作系统
- **NFR-18**: 支持主流浏览器（Chrome、Firefox、Edge、Safari）
- **NFR-19**: 支持不同屏幕分辨率
- **NFR-20**: 支持移动端访问

## 4. 系统需求

### 4.1 技术架构需求
- **SR-1**: 采用B/S架构，前端使用HTML/CSS/JavaScript
- **SR-2**: 后端使用Python Flask框架
- **SR-3**: 数据库存储使用SQLite
- **SR-4**: AI处理使用spaCy、scikit-learn等库

### 4.2 硬件环境需求
- **SR-5**: 最低配置：Intel i5处理器，8GB内存，500MB硬盘空间
- **SR-6**: 推荐配置：Intel i7处理器，16GB内存，1GB硬盘空间
- **SR-7**: 网络环境：稳定互联网连接（用于下载依赖包）

### 4.3 软件环境需求
- **SR-8**: 操作系统：Windows 10/11, macOS 10.14+, Ubuntu 18.04+
- **SR-9**: Python版本：3.8-3.13
- **SR-10**: 浏览器：Chrome 80+, Firefox 75+, Edge 80+, Safari 13+

## 5. 数据需求

### 5.1 数据实体定义

#### 5.1.1 文档实体
| 字段名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| id | 整数 | 文档唯一标识 | 主键，自增 |
| title | 字符串 | 文档标题 | 非空 |
| content | 文本 | 文档内容 | 可空 |
| file_path | 字符串 | 文件存储路径 | 非空 |
| file_type | 字符串 | 文件类型 | 非空 |
| file_size | 整数 | 文件大小（字节） | 非空 |
| tags | JSON | 文档标签 | 可空 |
| metadata | JSON | 元数据信息 | 可空 |
| created_at | 时间戳 | 创建时间 | 非空 |
| updated_at | 时间戳 | 更新时间 | 非空 |

#### 5.1.2 知识节点实体
| 字段名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| id | 整数 | 节点唯一标识 | 主键，自增 |
| name | 字符串 | 节点名称 | 非空 |
| node_type | 字符串 | 节点类型 | 非空 |
| description | 文本 | 节点描述 | 可空 |
| properties | JSON | 节点属性 | 可空 |
| frequency | 整数 | 出现频率 | 默认1 |
| created_at | 时间戳 | 创建时间 | 非空 |

#### 5.1.3 知识边实体
| 字段名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| id | 整数 | 边唯一标识 | 主键，自增 |
| source_id | 整数 | 源节点ID | 非空，外键 |
| target_id | 整数 | 目标节点ID | 非空，外键 |
| relation_type | 字符串 | 关系类型 | 非空 |
| weight | 浮点数 | 关系权重 | 默认1.0 |
| properties | JSON | 关系属性 | 可空 |
| created_at | 时间戳 | 创建时间 | 非空 |

#### 5.1.4 用户行为实体
| 字段名 | 类型 | 描述 | 约束 |
|--------|------|------|------|
| id | 整数 | 行为唯一标识 | 主键，自增 |
| user_id | 字符串 | 用户标识 | 非空 |
| document_id | 整数 | 文档ID | 可空，外键 |
| action_type | 字符串 | 操作类型 | 非空 |
| duration | 整数 | 操作时长（秒） | 可空 |
| details | JSON | 详细信息 | 可空 |
| timestamp | 时间戳 | 操作时间 | 非空 |

### 5.2 数据流分析

#### 5.2.1 文档处理数据流
1. 用户上传文档
2. 系统接收并存储文件
3. AI引擎处理文档内容
4. 提取关键词和实体
5. 生成文档标签和元数据
6. 存储到数据库

#### 5.2.2 知识图谱构建数据流
1. 从文档中提取知识点
2. 识别知识点之间的关系
3. 构建知识节点和边
4. 计算关系权重
5. 存储知识图谱数据
6. 提供可视化接口

#### 5.2.3 推荐系统数据流
1. 收集用户行为数据
2. 分析用户偏好
3. 计算文档相似度
4. 生成推荐列表
5. 提供推荐结果
